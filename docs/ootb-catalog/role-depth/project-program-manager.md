# Project Manager / Program Manager

## 1. Role summary

Owns timelines, dependencies, status reporting, risk escalation. PM Twin produces project-status / dependency-risk / blocker-escalation briefs. Team-level aggregation only. No name-and-shame.

## 2. Common titles

Project Manager · Senior PM · Program Manager · PMO Lead · Technical Program Manager.

## 3. Likely reports to

COO · PMO Director · Engineering Manager (TPM).

## 4. Possible reports to

CTO (TPM) · Head of Product (in Product orgs).

## 5. Likely direct reports

None at IC level.

## 6. Possible direct reports

Junior PM (in PMO).

## 7. Dotted-line relationships

Cross-functional owners.

## 8. Cross-functional partners

Product Manager · Engineering Manager · Design · CS · Sales.

## 9. External collaborators

Vendor PMs · customer PMs (for implementation projects).

## 10. Core responsibilities

Project plans · status reporting · dependency tracking · risk escalation · cross-team coordination.

## 11. Common decisions

Timeline adjustments · resource asks · escalation timing.

## 12. Common meetings

Weekly status · executive review · standup attendance · retros.

## 13. Common documents / artifacts

Project plan · status report · dependency map · risk log.

## 14. Common metrics / KPIs

On-time delivery · scope adherence · dependency burn-down. Team-level only.

## 15. Common tools

Jira · Asana · Monday · Smartsheet · Notion · Calendar · Slack / Teams.

## 16. Common workflows

- Project Status Report
- Dependency Risk Review
- Cross-Functional Blocker Escalation
- Timeline Change Brief

## 17. Approval authority

Project-level scope decisions within scope.

## 18. Approval dependencies

Timeline changes → executive sponsor; resource asks → manager.

## 19. Delegated authority

Project meeting facilitation.

## 20. Never-default permissions

Reassigning work without authority · changing deadlines without approval · reporting performance as employee scoring · cross-team visibility framing as surveillance.

## 21. Digital Twin day-one capabilities

Status report · dependency-risk brief · blocker escalation draft.

## 22. First-week aha moments

1. **Project Status Snapshot** — weekly status.
2. **Dependency Risk Map** — cross-team blockers.
3. **Cross-Functional Blocker Escalation** — escalation draft.
4. **Timeline Change Brief** — proposed change + impact.
5. **Risk Log Update** — weekly.

## 23. Safe fallback without connectors

Status template · risk-log template.

## 24. Connector implications

Slack · Project Tracker (Jira / Linear) · Calendar.

## 25. DMW / Memory Wallet scope notes

Project-scoped · team-scoped. Forbidden: individual performance signals beyond commitment data.

## 26. Governed context envelope notes

- `object_type`: PROJECT_OPERATING_MODEL
- `policy_purpose`: SCOPED_PROJECT_COORDINATION
- `permission_defaults`: READ_FIRST_PROJECT_SCOPE
- `sensitivity_level`: MEDIUM
- `forbidden_consumers`: TEAM_SHAME_METRICS · INDIVIDUAL_SCORING

## 27. Collaboration map

**Upward**: COO / PMO Director. **Downward**: project participants. **Peer**: PM · EM · Design. **Cross-functional**: all dependencies. **External**: vendor / customer PMs. **Approval path**: timeline change → exec sponsor; resource ask → manager. **Escalation**: cross-functional crisis → COO / EM.

## 28. Risks and guardrails

Team-shame metrics · name-and-shame · employee scoring · cross-team surveillance framing.

## 29. Industry / company-size variants

- **Startup**: PM doubles as Chief of Staff.
- **SMB / Mid-market**: dedicated PM per area.
- **Enterprise**: PMO + TPMs + Program Managers.
- **Regulated**: project audit + compliance gating.
